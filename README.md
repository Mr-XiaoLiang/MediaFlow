# Media Flow

原本我打算是做一个就像抖音那样的本地视频播放器，事实上也确实做到了。

你可以看[这里](./vision/README.md)。

但是，感觉应该不只是如此，视频与图片都做了，为什么不顺便做个音频的呢？

所以整理了一下模块，再加上音频的部分。

## Vision

视觉模块，也就是原本的 MediaFlow，现在它被改为了视觉模块。

它功能没有变化，提供了以下功能：

- 本地视频、图片按文件夹添加
- 隐私模式的视频与图片独立管理
- 类短视频的播放模式（播放页面上下滑动即可切换）
- 完全的手势处理，播放页面允许通过手势快进快退，并且支持变速快进
- 快速的预加载能力，接近无感的视频切换
- 瀑布流的视频列表页面
- 播放页面支持侧边栏快速定位
- 播放页面支持宫格的视频切换
- 完全的横竖屏兼容，允许播放过程中切换横竖屏
- 多文件夹的归档功能，提供额外的归档管理页面
- 覆盖全面的UI显示状态设置
- 视频字幕的支持
- 流式图片展示，无缝浏览图片，方便漫画查看

  - 播放列表（Playlists）：用户自建、最近添加、播放最多等
- 最近播放/常用功能：置顶展示最近听过的 3-5 首歌，以及“我喜欢”和“随机播放全部”按钮。
- 今日发现（本地版）：利用算法从本地曲库抽取的随机组合（如“遗忘的角落”——很久没听的歌）。
- 最近添加：展示最新扫描到手机里的音频，方便用户快速收听新下载的资源
- 智能推荐（本地版）
  - 基于“记忆”的推荐：
    - 时光机：推荐去年此时、上个月此时播放频率最高的歌曲。
    - 播放热榜：根据本地播放计数器（Play Count）生成 Top 50。
  - 基于“元数据”的相似性：
    - 同风向标：如果用户正在听某首歌，自动生成一个“同歌手”或“同风格”的临时队列。
    - 专辑连播：检测到用户常听某专辑中的多首歌，主动推荐该专辑的完整收听。
  - 基于“场景”的自动化：
    - 时间感应：早晨推荐节奏轻快的歌，深夜推荐 Bpm 较低或标记为“轻音乐”的音频。
    - 设备感应：插入耳机或连接蓝牙时，弹出“继续播放”或根据该耳机下常听的歌单进行推荐。
